February 2, 2010 ~ Development on Data Crow 4.0 has been started

The Data Crow 3.0 version has and is serving its purpose, but... it's time to move on! Data Crow 4 will pick up the pieces left behind by Data Crow 3. Focus? Increased performance and memory usage reduction. Is that all? No, apart from the necessary platform changes Data Crow 4 will include features that makes product even more flexible.

What will this mean for the current Data Crow 3 product? Essentially no new development will be done on this version; only maintance releases will be created.

Data Crow 4 will include the following fundamental changes:

New functionality

  • Multiple files per item (including the file meta data).
  • Multiple images per item (there will be no dependecy on the fields).
  • Global item importer + file browser.
  • Book module upgrade to support documents, essays, etc.
  • Server - client mode. Data Crow will be able to run in server mode. Not only will this allow the web server to fully operate as a standalone service but this will also allow the full software clients to connect to one single server.

    Peformance and memory optimizations:

  • Intelligent data layer. Items will no longer be loaded on startup. The view functionality has already been adopted to support this. However the data layer still loads all items on startup. This is the correct way when dealing with a small amount of items. However, people having thousands of authors and other items in the database have noticed that this causes memory and performance issues. With the new data layer and the adopted view only the items currently visible in the view will be loaded.
